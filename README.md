# plnt-site

The showcase site for [**plnt**](https://github.com/plnt-work/plnt) — a personal local native twin.

Built with [Astro](https://astro.build) + [Starlight](https://starlight.astro.build) for the docs portal. Static, zero-JS landing, terminal-green brand, mono-only typography. Deploys to any static host.

## Layout

```
src/
  pages/
    index.astro          # landing — sprout logo, spawn-tree hero, key choices, intent flow, install
    architecture.astro   # 9-section deep dive — AgentSpec, blackboard, sandbox ladder, real events.jsonl
    roadmap.astro        # v0 ships, 12-week plan, v0 → v0.1 punch list
    resources.astro      # docs + source + reading list
    terms.astro, privacy.astro
  content/docs/docs/     # Starlight docs portal at /docs/
    getting-started/
    concepts/
    reference/
  components/            # SpawnTree, CreamCard, MetaRow, SectionRule, KeyChoice, FlowStep, ...
  layouts/Layout.astro
  styles/global.css      # palette, typography, components
  styles/starlight-overrides.css
  assets/logo-mono.svg
  data/hero-ascii.txt    # generated by scripts/gen-ascii.mjs
scripts/
  gen-ascii.mjs          # regenerate the ASCII texture
```

## Develop

```bash
npm install
npm run dev           # http://localhost:4321
npm run build         # → dist/
npm run preview       # serve dist/
node scripts/gen-ascii.mjs   # regenerate hero texture
```

## Deploy

Drop `dist/` on Vercel, Netlify, Cloudflare Pages, or GitHub Pages. Update the `site:` URL in `astro.config.mjs` once a real domain is wired.
