The **blackboard** is the audit story. Every observable change of state in the
system — every intent, every spawn, every model call, every tool call — is one
JSONL line under `$PLNT_HOME/runs/<run_id>/events.jsonl`.

If `cat` can't see it, it doesn't exist as state.

## Why JSONL

- **Append-only.** No locking, no corruption window. Every writer flushes one line at a time.
- **`cat`-able.** `cat`, `grep`, `jq`, `awk` are all the admin tools you need.
- **Replayable.** Any UI is a function of `events.jsonl` — the TUI, the dashboard, future replays.
- **Cross-process.** Each agent writes its own events; the orchestrator interleaves them in event-time order on read.

## Audit queries

Find every cloud backend call across all runs:

```bash
grep -rF '"backend":"cloud"' ~/.plnt/runs/
```

Tool histogram for one run:

```bash
jq -r 'select(.kind=="tool_call") | .payload.tool' \
  ~/.plnt/runs/r-XXXXXXXX/events.jsonl | sort | uniq -c
```

Reconstruct the spawn tree:

```bash
jq -r 'select(.kind=="spawn") | "\(.agent_id) ← \(.payload.parent_id // "ROOT") [\(.payload.role)]"' \
  ~/.plnt/runs/r-XXXXXXXX/events.jsonl
```

Every wall-clock interval an agent took longer than 5s to respond:

```bash
jq -s 'group_by(.agent_id) | map(select(length>1)) |
       map({agent:.[0].agent_id, gaps: [range(1; length) as $i |
            (.[$i].ts - .[$i-1].ts) | select(.>5)]})' \
  ~/.plnt/runs/r-XXXXXXXX/events.jsonl
```
