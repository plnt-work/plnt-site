## Layers

| Layer                | Component                                    | Owns                                        |
|----------------------|----------------------------------------------|---------------------------------------------|
| CLI + Python API     | `plnt/cli.py`                                | `plnt deploy`, `plnt playground …`          |
| Playground gateway   | `plnt/playground/`                           | OpenAI-compat REST, SSE, `RuntimeAdapter`   |
| Helm charts          | `plnt/charts/{playground-api, vllm-runtime}` | One chart per runtime + one for the gateway |
| Temporal workflows   | `plnt/workflows/`                            | Deploy saga, canary rollout, batch infer    |
| Runtime adapters     | `plnt/playground/backends.py`                | vLLM / TGI / TRT-LLM / SGLang, mock         |
| CRD + operator       | `plnt/operators/`                            | `InferenceModel` CRD, kopf controller       |
| Benchmarks           | `plnt/bench/`                                | TTFT / TPOT / tokens-per-sec/GPU            |

## The playground API

Serves the OpenAI wire format from a single URL — `playground.plnt.work`. The site at `plnt.work` embeds a chat panel that posts to it, so a visitor can talk to whatever's deployed on the cluster without leaving the marketing page.

Model list is **configuration, not code** — the Helm chart ships a ConfigMap that the pod reads at startup. `helm upgrade` with a new list restarts the pod and picks up the change. There is no runtime `POST /models` — model lifecycle is a Helm concern, not a REST call, which keeps the audit trail in `helm history` where cluster operators expect it.

Backends behind the gateway:

- `mock` — deterministic echo, self-contained. Used on kind (no GPU) and in tests. Streams word-by-word via SSE so the UI behaves identically to a real model.
- `http` — proxies to any OpenAI-compatible upstream. vLLM, TGI (`--openai-api`), SGLang, and llama.cpp server all satisfy this contract.

## The deploy path

1. `kubectl apply -f examples/llama-3.1-70b-instruct.yaml` — an [InferenceModel](/docs/concepts/inferencemodel/) resource lands in the cluster.
2. The [kopf operator](/docs/concepts/deploy-saga/) sees the create event and starts a Temporal `DeployModelWorkflow`.
3. The workflow runs the [five-step saga](/docs/concepts/deploy-saga/) — validate, pull, helm install canary, smoke, promote. Compensation on failure.
4. Once promoted, the model is reachable at `<name>.<ns>.svc.cluster.local:8000/v1/chat/completions`.
5. The playground gateway routes external requests to it based on `req.model`.

## Design invariants

- **Every runtime speaks OpenAI wire format at the pod boundary.** Existing SDKs work.
- **The saga is a Temporal workflow, not a bash pipeline.** State is durable, resumable, and inspectable.
- **Helm + K8s + Temporal are portable.** Runs on kind, GKE, EKS, AKS, on-prem — same YAML.
- **The operator is small enough to read in one sitting.** Every state transition emits a Kubernetes event.

See [Runtime adapter](/docs/concepts/runtime-adapter/) for the code shape shared across backends, and [API contract](/docs/reference/api-contract/) for the exact wire format.
