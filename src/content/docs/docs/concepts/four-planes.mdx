Plnt is split into four planes with **hard responsibility boundaries**. Solid arrows
are control flow. Dashed arrows are inference calls. The `AgentSpec` is the
only object that crosses **Control → Execution**.

## Surface

> Task panels on your devices.

The Surface is how intents land and results return. It is the thinnest plane and
holds nothing but routing.

- **`plnt/surface/server.py`** — FastAPI server. `/v1/intents`, `/v1/runs`, `/v1/runs/{id}/stream` (SSE), `/v1/skills`, `/v1/health`, `/v1/system`.
- **`plnt/cli.py`** — `plnt` entrypoint. `submit`, `up`, `runs`, `tail`, `skills`, `monitor`.
- **`plnt-tui/`** — Go / Bubble Tea TUI client. Streams the blackboard over SSE; routes events by `run_id`.

mTLS is a stub today (loopback only). The roadmap calls for `plnt enroll` to make
"your phone reaches the server" real.

## Control

> One resident process — the brain.

The Control plane is where intents become plans become spawns. Today the orchestrator
materialises a planner per intent; v0.1 makes it a long-lived resident.

- **`orchestrator.py`** — `start_run`, path harvesting + injection, DAG dispatch.
- **`planner_llm.py`** — Kimi-style triage → plan → DAG.
- **`triage.py` · `clarify.py`** — classify intent; deterministic Q&A for ambiguous asks.
- **`dag.py`** — express sub-task dependencies.
- **`parallel.py`** — fan sub-tasks out to ephemeral agents.
- **`skills.py` · `skill_installer.py`** — markdown skill bundles; can pull from `anthropic/skills` and 4 other catalogs.
- **`budget.py`** — hard token/time/depth caps.
- **`acc.py`** — ACC (Anomalous Control Coordinator). Watches for loops, fanout, ping-pong; can issue kills.
- **`synthesizer.py`** — combine sub-task outputs into one markdown file.

## Execution

> Ephemeral micro-agents in sandboxes.

Each `AgentSpec` becomes a sandboxed process running the RLM loop. Two tools, ever.

- **`spec.py`** — the `AgentSpec` Pydantic model. See [AgentSpec](/docs/concepts/agentspec/).
- **`blackboard.py`** — the JSONL append-only event store. See [Blackboard](/docs/concepts/blackboard/).
- **`runner.py`** — RLM loop. Parses `TOOL: {...}` / `FINAL: ...` from model output, dispatches `search`/`execute`, appends every event to the blackboard. SIGTERM handler ensures a `transcript-summary` lands even on watchdog kill.
- **`sandbox/process.py`** — rung 0. Subprocess + rlimits + watchdog.
- **`sandbox/docker.py`** — rung 1. CPU/memory caps, parallel-safe via labelled containers.
- **`tools/search.py` · `tools/execute.py`** — the two tools.

See the full ladder → [Sandbox ladder](/docs/concepts/sandbox-ladder/).

## Compute

> Local inference, optionally federated.

The Compute plane is an OpenAI-compatible router. The planner emits a hint
(`small` / `deep` / `auto`); the router decides which backend serves it.

- **`router.py`** — speaks OpenAI-compat. Has an offline echo fallback so the
  full test suite runs with no LLM at all.
- **`backend_picker.py`** — picks local / cloud / echo per call. See [env vars](/docs/reference/env-vars/) for the decision inputs.

<Aside type="note">
Federate the **inference**, not the agent state. Multi-device inference is solved
(exo, vLLM, distributed Ollama). Multi-device agent state is a debugging hellscape —
plnt keeps state on the one machine that owns the planner.
</Aside>

## Hard rules

1. **`AgentSpec` is the only Control → Execution object.** Nothing else crosses.
2. **Two tools only: `search`, `execute`.** No third tool, ever.
3. **Blackboard is the audit story.** Every observable event is one JSONL line.
4. **The framework owns memory, skills, and the sandbox.** The model is a swappable backend.
