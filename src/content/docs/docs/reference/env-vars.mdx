## Storage

| Variable | Default | Purpose |
|---|---|---|
| `PLNT_HOME` | `~/.plnt` | Where runs, skills, episodic memory, and the identity file live. |
| `PLNT_REQUIRED_PATH` | unset | Path that must be mounted to choose the local backend (e.g. external SSD with the models). |

## Local backend (Ollama, MLX, LM Studio, …)

| Variable | Default | Purpose |
|---|---|---|
| `PLNT_LOCAL_URL` | `http://127.0.0.1:11434` | OpenAI-compat local endpoint. |
| `PLNT_PLANNER_MODEL` | `llama3.2:3b` | Small/local model id (planner, triage, scouts). |
| `PLNT_DEEP_MODEL` | `llama3.1:8b` | Deep/local model id (synthesizer, hard tasks). |

## Cloud fallback

| Variable | Default | Purpose |
|---|---|---|
| `PLNT_CLOUD_URL` | unset | OpenAI-compat cloud endpoint. |
| `PLNT_CLOUD_API_KEY` | unset | Cloud API key. |
| `PLNT_CLOUD_SMALL_MODEL` | provider default | Cloud small model id. |
| `PLNT_CLOUD_DEEP_MODEL` | provider default | Cloud deep model id. |

## Federated inference (exo)

| Variable | Default | Purpose |
|---|---|---|
| `PLNT_DEEP_URL` | unset | Point at an exo cluster for federated heavy calls. |

## Backend picker — the actual decision tree

`plnt/compute/backend_picker.py` runs this order:

1. **Forced override** — if the call passes `force=local|cloud|offline`, use it.
2. **Local** — if `PLNT_REQUIRED_PATH` is mounted (or unset) **and** `PLNT_LOCAL_URL` is TCP-reachable → local.
3. **Cloud** — if `PLNT_CLOUD_URL` + `PLNT_CLOUD_API_KEY` are set **and** the endpoint is reachable → cloud.
4. **Offline echo** — otherwise. Deterministic; the test suite uses this path.

Every decision lands in `events.jsonl` with a `backend` field, so `grep -F '"backend":"cloud"'`
across `~/.plnt/runs/` is a real audit query.

<Aside type="tip">
The simplest config: install Ollama, pull a small model, and set nothing. Plnt picks
the local backend automatically because Ollama listens on the default URL.
</Aside>
